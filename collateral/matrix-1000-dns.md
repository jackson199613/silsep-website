# Matrix 1000: Distributed Isolation DNS Deployment Guide

This document outlines the required DNS records to deploy **SILSEP's** satellite domains on Google Workspace and Microsoft 365. This configuration is critical for the "Distributed Isolation" strategy to ensure high deliverability and protection of the main brand domain.

## 1. Google Workspace Cluster
**Domains:** `aqua-silsep.com`, `silsep-vn.com`

### Verification (Required for Initial Setup)
| Type | Host | Value | Note |
|------|------|-------|------|
| TXT  | @    | google-site-verification=[Your-Unique-Key] | Retrieve from Google Admin Console |

### MX Records (Email Routing)
| Priority | Host | Points to |
|----------|------|-----------|
| 1        | @    | ASPMX.L.GOOGLE.COM. |
| 5        | @    | ALT1.ASPMX.L.GOOGLE.COM. |
| 5        | @    | ALT2.ASPMX.L.GOOGLE.COM. |
| 10       | @    | ALT3.ASPMX.L.GOOGLE.COM. |
| 10       | @    | ALT4.ASPMX.L.GOOGLE.COM. |

### Deliverability & Security (SPF/DMARC)
| Type  | Host | Value |
|-------|------|-------|
| SPF   | @    | `v=spf1 include:_spf.google.com ~all` |
| DMARC | _dmarc | `v=DMARC1; p=quarantine; rua=mailto:dmarc-reports@silsep.com` |

## 2. Microsoft 365 Cluster
**Domain:** `silsep-mekong.com`

### Verification (Required for Initial Setup)
| Type | Host | Value | Note |
|------|------|-------|------|
| TXT  | @    | MS=ms[Your-Unique-Number] | Retrieve from Microsoft 365 Admin Center |

### MX Records (Email Routing)
| Priority | Host | Points to |
|----------|------|-----------|
| 0        | @    | silsep-mekong-com.mail.protection.outlook.com. |

### CNAME Records (Required for M365 Discovery)
| Host      | Points to |
|-----------|-----------|
| autodiscover | autodiscover.outlook.com. |

### Deliverability & Security (SPF/DMARC)
| Type  | Host | Value |
|-------|------|-------|
| SPF   | @    | `v=spf1 include:spf.protection.outlook.com -all` |
| DMARC | _dmarc | `v=DMARC1; p=quarantine; rua=mailto:dmarc-reports@silsep.com` |

---

## 3. Implementation Steps (Namecheap/GoDaddy)
1. Login to your Domain Registrar (Namecheap/GoDaddy).
2. Navigate to **Advanced DNS** or **DNS Management**.
3. **Delete existing MX records** (the eforward ones currently pointing to registrar-servers.com).
4. **Add the new MX records** from the tables above.
5. **Add the TXT/SPF records**. Ensure there is only ONE SPF record per domain.
6. **Wait for propagation** (usually 1-2 hours).
7. Return to Google/Microsoft Admin Consoles to "Verify Domain."
